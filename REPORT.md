## 1. Описание предметной области

### Военная техника: Наземные боевые машины

В данной лабораторной работе рассматривается иерархия классов боевой наземной техники. 
Основу иерархии составляет абстрактный класс **Танк**, от которого наследуются:
- **БМП** (Боевая Машина Пехоты) — для перевозки и поддержки пехоты
- **ЗРК** (Зенитный Ракетный Комплекс) — для поражения воздушных целей

### Почему выбрана эта предметная область:
- Наглядно демонстрирует наследование (общие свойства танка + специфика подклассов)
- Богатый набор методов для каждого класса
- Полиморфизм естественно применяется при работе с разными типами техники

## 4. Особенности реализации

### 4.1 Абстрактный класс
Класс Tank является абстрактным благодаря чисто виртуальным методам:
```cpp
virtual void info() const = 0;
virtual void attack() const = 0;
```
Невозможно создать экземпляр Tank напрямую.

### 4.2 Инкапсуляция
- Закрытые данные (`ammo`, `fuel`) доступны только через геттеры/сеттеры
- Защищенные данные (`name`, `armor`, `speed`) доступны в классах-наследниках
- Внешний код использует только открытый интерфейс

### 4.3 Наследование
- BMP и ZRK наследуют все методы класса Tank
- Каждый дочерний класс имеет 3 конструктора
- Переопределены виртуальные методы `info()` и `attack()`

### 4.4 Полиморфизм
- Объекты разных классов хранятся в массиве `vector<Tank*>`
- Вызов `info()` и `attack()` через указатель на базовый класс
- Каждый объект вызывает свою версию метода

## 5. Демонстрация принципов ООП

### 5.1 Инкапсуляция (Encapsulation)
```cpp
BMP bmp;
bmp.setName("БМП-2");
bmp.setArmor(150);
bmp.setSpeed(65);
bmp.setAmmo(300);  // Доступ через сеттер, не напрямую
```
Доступ к закрытым полям осуществляется только через открытые методы.

### 5.2 Наследование (Inheritance)
```cpp
ZRK zrk("Стрела-10", 100, 60);
zrk.refuel(50);      // Унаследованный метод из Tank
zrk.reload(100);     // Унаследованный метод из Tank
```
Класс ZRK наследует методы `refuel()` и `reload()` от Tank.

### 5.3 Полиморфизм (Polymorphism)
```cpp
std::vector<Tank*> army;
army.push_back(new BMP("БМП-3", 180, 70));
army.push_back(new ZRK("Тор-М2", 120, 50));

for (size_t i = 0; i < army.size(); i++) {
    army[i]->info();    // Вызывается правильная версия
    army[i]->attack();  // для каждого типа техники
}
```
Один и тот же вызов метода приводит к разному поведению.

## 6. Проверочные требования

| Требование | Статус | Примечание |
|------------|--------|------------|
| Базовый класс абстрактный | ✅ | `virtual void info() = 0;` |
| ≥2 дочерних класса | ✅ | BMP, ZRK |
| ≥2 членов данных в каждом классе | ✅ | У всех по 5+ |
| ≥2 собственных методов | ✅ | embarkDemolition, setSmokeScreen и др. |
| ≥2 унаследованных методов | ✅ | refuel, reload, getPower |
| ≥2 переопределенных методов | ✅ | info, attack |
| Дочерние классы имеют ≥3 конструкторов | ✅ | По 3 конструктора у BMP и ZRK |
| Доступ через геттеры/сеттеры | ✅ | Все поля закрытые |
| Работа через родительский тип | ✅ | `vector<Tank*>` |
| Каждый класс в отдельном файле | ✅ | Tank.h/cpp, BMP.h/cpp, ZRK.h/cpp |

## 7. Выводы

В ходе лабораторной работы были реализованы:

1. **Абстрактный базовый класс Tank** — определяет общий интерфейс для всех боевых машин
2. **Класс BMP** — демонстрирует работу с десантом и дымовыми завесами
3. **Класс ZRK** — демонстрирует работу с радаром и захват воздушных целей

**Инкапсуляция** обеспечивает сокрытие данных: закрытые поля `ammo` и `fuel` доступны только через геттеры/сеттеры.

**Наследование** позволяет классам BMP и ZRK использовать общую функциональность класса Tank без дублирования кода.

**Полиморфизм** реализован через виртуальные методы: массив указателей на Tank содержит объекты разных типов, и при вызове `info()`/`attack()` выполняется нужная версия.
